## 🐉 نبذة عن DragonClaw

> **ربما هو أفضل عميل سطح مكتب لـ OpenClaw متوفر اليوم.**

**DragonClaw** هو عميل سطح مكتب مصمم خصيصًا لـ [OpenClaw](https://example.com/openclaw)، ومطوّر أعلى واجهة WebSocket الرسمية لـ OpenClaw. بُني على حزمة تقنية حديثة **Electron + Vue 3 + Arco Design**، ويأتي مُرفقًا بكل ما يلزم OpenClaw من بيئات تشغيل — **ثبّت وشغّل فورًا**. إذا كان OpenClaw مُثبّتًا على جهازك مسبقًا، فإن DragonClaw سيكتشفه تلقائيًا ويعيد استخدامه، بدون أي تكلفة ترحيل.

DragonClaw ليس مجرد «قشرة رسومية» لـ Gateway؛ بل يقدّم مجموعة من **الإمكانات المبتكرة على مستوى المنتج**:

- 🧠 **إدارة الوكلاء** — إدراج وإنشاء وتحديث وحذف الوكلاء، وقراءة/كتابة ملفات مساحة العمل لكل وكيل.
- 🧩 **مركز المهارات** — تصفّح المهارات الموصى بها، والبحث في الكتالوج الكامل، والتثبيت/الإزالة بنقرة واحدة.
- 💬 **محادثات متعددة** — تشغيل عدة جلسات بالتوازي مع تجميع ذكي وتثبيت للأعلى وعلامات غير مقروء وسجل كامل.
- 📁 **مساحات عمل على مستوى الجلسة** — حدّد مجلد العمل للجلسة الحالية مباشرة من واجهة المحادثة، فتبقى كل مخرجات OpenClaw محصورةً داخل مساحة العمل تلك دون أن تلوّث أي مكان آخر.
- 🌳 **مهام فرعية** — تصوير المهام الفرعية والجلسات الفرعية التي يُنشئها الوكيل الرئيسي في الوقت الفعلي.
- 📚 **تهيئة النماذج** — دعم مدمج للمزوّدين الرئيسيين: OpenAI و Anthropic و DeepSeek و Grok و OpenRouter و Groq و Moonshot و Alibaba و Ollama و ModelBus، إضافةً إلى مزوّد مخصّص.
- 🏪 **متجر Agents** — اختر الوكلاء المناسبين عبر الإنترنت، وثبّتهم مع المهارات التي يعتمدون عليها بنقرة واحدة محليًا.
- 📋 **لوحة كانبان** — لوحة مهام لتنظيم العمل عبر الوكلاء.
- 📜 **اللوحة السجلية** — اعرض سجلات Gateway الحيّة مباشرة من التطبيق.
- ⚙️ **إعدادات موحّدة** — الأساسيات والبوابة والوكلاء والجلسات والأدوات والأمان في لوحة واحدة.
- 🔄 **كشف التحديثات** — فحص تلقائي عند بدء التشغيل لـ DragonClaw نفسه و OpenClaw و Node.js وغيرها.
- 🌐 **الوصول عن بُعد** — اتصل بـ OpenClaw Gateway داخل شبكتك المحلية أو على خادم بعيد، بمصادقة عبر Token أو كلمة مرور.

> 📖 للاطّلاع على شرح معمّق لكل ميزة (المحادثات، مساحات العمل، المهام الفرعية، تهيئة النماذج، متجر Agents، كشف التحديثات، الوصول عن بُعد) راجع قسم [🔍 شرح المميزات](#-شرح-المميزات) أدناه.

## 🔍 شرح المميزات

فيما يلي نظرة أقرب على أبرز إمكانات DragonClaw وحالات استخدامها.

### 💬 المحادثات (Conversation)

لوحة المحادثات هي قلب DragonClaw. إنها ليست مجرد نافذة دردشة، بل **نظام تعاون متعدد الجلسات بالكامل**:

- **جلسات متوازية** — افتح عدة محادثات في آنٍ واحد، كل واحدة مربوطة بوكيل أو مهمة مختلفة. بدّل من الشريط الجانبي دون أي تداخل.
- **تجميع ذكي** — تُنظَّم الجلسات التاريخية تلقائيًا في مجموعات *اليوم* و*أمس* و*هذا الأسبوع* و*أقدم*.
- **تثبيت وغير مقروء** — يمكن تثبيت الجلسات المهمة، وتظهر الجلسات غير المقروءة بنقطة حمراء.
- **بث المخرجات** — ردود النموذج تُعرض في الزمن الحقيقي، مع تصوير لمسار التفكير ونتائج استدعاء الأدوات داخل النص.
- **الرجوع في السجل** — مرّر للأعلى لتحميل رسائل أقدم بكسل؛ لا يُحذف شيء.
- **إجراءات الرسائل** — نسخ واقتباس وحذف و«ضغط» الجلسات كلها متاحة محليًا.
- **مستوى التفكير** — اختر شدة الاستدلال (عالٍ / متوسط / منخفض) من الشريط العلوي للموازنة بين السرعة والجودة.
- **تصوير المهام الفرعية** — تظهر المهام الفرعية والوكلاء الفرعيون كروابط مدمجة داخل مجموعة الرسائل ذاتها، مع الوكيل المسؤول والحالة الحيّة.

#### 📁 مساحات العمل على مستوى الجلسة (Session-level Workspace)

> **مساحات العمل في DragonClaw تعمل على مستوى الجلسة، ولا تلوّث أي مكان آخر.** يمكنك ربط مجلد عمل مخصّص للجلسة الحالية من داخل واجهة المحادثة، وتحتفظ كل جلسة بمساحة عمل مستقلة — فكل ما يقرأه OpenClaw أو يكتبه أو ينتجه داخل هذه الجلسة يبقى محصورًا في مساحة العمل هذه، دون تسرّب إلى مجلدات أخرى أو إلى النطاق العام.

تُربط مساحة العمل بـ **الجلسة الحالية**، لا بالوكيل عالميًا؛ لذلك يمكن أن تستخدم جلستان مع نفس الوكيل مجلدين مختلفين تمامًا.

- **الإعداد من واجهة المحادثة** — اضغط زر «مساحة العمل» في شريط الإدخال لاختيار مجلد، وسيُربط فورًا بالجلسة النشطة.
- **نطاق لكل جلسة** — يُخزَّن المسار على مفتاح الجلسة (`projectSpace`)، فعند التبديل بين الجلسات تعرض كل واحدة مجلد عملها، لا مجلدًا عالميًا مشتركًا.
- **سياق يتبع الجلسة** — كل عمليات القراءة/الكتابة للملفات وتنفيذ الأوامر داخل تلك الجلسة تستخدم مساحة العمل كجذر، فلا يحتاج الوكيل إلى سؤال «أين أحفظ؟» مرارًا.
- **فتح وإعادة ربط** — مساحة العمل الحالية ظاهرة دائمًا في شريط الإدخال؛ انقر لفتحها في مدير ملفات النظام، أو أعد ربطها بمجلد جديد متى شئت.
- **تنبيه للحالة الفارغة** — إذا لم تُحدَّد مساحة عمل، يضيء شريط الإدخال لتنبيهك ومنع أي إجراء خاطئ.

#### 🌳 المهام الفرعية (Sub-tasks)

عندما يُنشئ الوكيل الرئيسي مهامًا فرعية أو وكلاء فرعيين، يتتبّعهم DragonClaw تلقائيًا ويعرضهم بطريقتين متكاملتين:

- **عرض مدمج داخل الرسالة** — بطاقات المهام الفرعية تظهر أسفل الرسالة المُنشِئة مباشرة، مع الوكيل المُنفّذ وعنوان المهمة وحالتها.
- **ملخص في اللوحة الجانبية اليمنى** — افتح اللوحة اليمنى لرؤية جميع المهام الفرعية للجلسة الحالية وحالة إنجازها دفعةً واحدة.
- **تزامن حيّ للحالة** — انتقالات الحالة (pending → done) تتحدّث في الزمن الحقيقي مع دفع Gateway للأحداث.

### 🧠 تهيئة النماذج (Model Configuration)

تهيئة النماذج من أقوى جوانب DragonClaw. في حين يُجبرك OpenClaw التقليدي على تعديل `config.json` يدويًا، يقدّم DragonClaw **تجربة إدارة نماذج رسومية بالكامل**، و**يُدمج جميع مزوّدي النماذج الرئيسيين افتراضيًا**:

- **مزوّدون رئيسيون مُدمجون** — OpenAI و Anthropic و DeepSeek و Grok و OpenRouter و Groq و Moonshot و Alibaba و Ollama و ModelBus، إضافةً إلى «Custom Provider» قابل للتخصيص بالكامل — جميعها مُهيّأة مسبقًا ولا تحتاج إلى أي إعداد يدوي.
- **إدارة موحّدة لعدة مزوّدين** — عشرات الموردين في شاشة واحدة، ويمكنك تبديل النموذج الرئيسي لأي جلسة في ثوانٍ.
- **عرض مُجمَّع حسب المزوّد** — تُجمَّع الإدخالات تلقائيًا حسب المزوّد مع الشعار وعدد النماذج وطول السياق.
- **بيانات وصفية غنية** — تُجلب من كتالوج `models.dev`: نافذة السياق والإمكانات المدعومة (أدوات، رؤية، …) وفئة السعر وغيرها.
- **إنشاء/تعديل/حذف بصري** — إضافة نموذج تحتاج إلى عدد قليل من الحقول (المعرّف، الاسم المعروض، الحد الأقصى للتوكنات…)، دون كتابة JSON.
- **عزل بيانات الاعتماد** — مفاتيح API و BaseURL معزولة لكل مزوّد فلا تتداخل فيما بينها.
- **النماذج المحلية مواطن من الدرجة الأولى** — Ollama مُهيّأة مسبقًا بـ `http://127.0.0.1:11434/v1`، والنماذج المحلية تعمل فورًا.
- **تبديل على مستوى الجلسة** — غيّر النموذج الرئيسي للمحادثة من شريطها العلوي، دون إعادة تشغيل Gateway.

### 🏪 متجر Agents

> **اختر الوكلاء المناسبين عبر الإنترنت، وثبّتهم مع المهارات التي يعتمدون عليها بنقرة واحدة محليًا — وداعًا لاستنساخ القوالب يدويًا من GitHub.**

يأتي DragonClaw مزوّدًا بـ **متجر Agents** مدمج يتيح لك تصفّح الوكلاء وتثبيتهم — إلى جانب المهارات التي يحتاجونها — من داخل التطبيق مباشرة:

- **كتالوج على الإنترنت** — كتالوج منسّق من الوكلاء الرسميين ومجتمع المستخدمين داخل التطبيق.
- **اختر الأنسب لعملك** — فلترة حسب السيناريو (تطوير، كتابة، أعمال مكتبية، تشغيل وصيانة، …) لاختيار الوكيل الأنسب لمهمتك، بدلًا من الاكتفاء بما هو مُثبّت محليًا.
- **تثبيت الوكلاء والمهارات بنقرة واحدة** — عند اختيار وكيل معيّن، يُثبَّت مع جميع المهارات التي يعتمد عليها تلقائيًا، فلا حاجة إلى حلّ التبعيات يدويًا واحدةً تلو الأخرى.
- **تخصيص محلي** — التثبيت بصري بالكامل: اختيار الصورة الرمزية، تسمية المعرّف، ربط مجلد العمل، وصف الشخصية، إلخ.
- **عرض «وكلائي»** — تبويب مخصّص يعرض كل ما أنشأته، جاهزًا للتعديل والحذف والاطّلاع.
- **تحديثات على الإنترنت** — حين يُتاح إصدار جديد لأحد الوكلاء المُثبّتين، يمكنك الترقية إليه بنقرة واحدة.

![متجر Agents](doc/assets/Agents-Store.png)

### 🔄 كشف التحديثات (Update Detection)

يتضمّن DragonClaw **مُحدِّثًا شاملًا للمكونات** يعمل **تلقائيًا عند بدء التشغيل**، فيوفّر عناء مقارنة أرقام الإصدارات بين المنظومات المختلفة:

- **فحص تلقائي عند البدء** — في كل مرة يبدأ فيها DragonClaw، يطلب واحد يسترجع أحدث إصدارات DragonClaw و OpenClaw و Node.js وغيرها تلقائيًا.
- **مصدر موثَّق بالتوقيع** — يمر عبر نقطة `api.versionCheck` في `config.json` بمعرّف `sign=dragonclaw`.
- **تنبيهات مرئية** — عند توفّر إصدار جديد، يعرض الشريط الجانبي وصفحة الإعدادات إشعارًا واضحًا مع رابط سجل التغييرات.
- **تنزيل وتثبيت داخل التطبيق** — التنزيل والتثبيت والاستبدال كلها من داخل التطبيق، دون حجب النافذة الرئيسية.
- **حماية من التراجع** — حالات فشل التنزيل أو عدم تطابق التوقيع لا تُفسد التثبيت الحالي، فلا تتعطّل الشاشة البيضاء.

### 🌐 الوصول عن بُعد (Remote Access)

سواء كانت OpenClaw Gateway تعمل على جهاز آخر داخل شبكتك المحلية أم على خادم بعيد أو جهاز داخلي أو NAS منزلي، فإن DragonClaw يتّصل بها بأمان ومرونة عبر **Token** أو **كلمة مرور**:

- **تبديل بنقرة واحدة** — اختر «Remote Mode» من القائمة أو بدّل من الشريط الجانبي، وستتصل بالبوابة المستهدفة في ثوانٍ.
- **مصادقة مرنة** — يدعم كلًا من المصادقة بـ Token وبكلمة المرور، لتلائم مختلف سياسات الأمان في البوابات.
- **مؤشّر حالة حيّ** — يعرض الشريط الجانبي *Connecting / Connected / Disconnected* في الزمن الحقيقي، مع زر إعادة اتصال يدوية.
- **تجربة موحّدة بين المحلي والبعيد** — بمجرد الاتصال، تعمل كل الميزات (الوكلاء، الجلسات، المهارات، الإعدادات، السجلات…) تمامًا كما في الوضع المحلي.
- **اختبار الاتصال** — قبل التبديل، يمكنك اختبار الوصول إلى المضيف الهدف لتفادي إضاعة الوقت بسبب إعداد خاطئ.
- **حفظ الإعدادات** — يُخزَّن العنوان البعيد والمنفذ و Token مشفّرًا محليًا، ويُستعاد تلقائيًا عند التشغيل التالي.

> 💡 إذا كان الجهاز البعيد بلا عنوان IP عام، يمكنك إقران DragonClaw مع Tailscale أو ZeroTier أو frp أو أي حل آخر للأنفاق/NAT traversal.

### 📋 لوحة كانبان (Kanban)

عندما تحتاج إلى تقسيم العمل عبر عدة وكلاء يعملون بالتوازي، تُفرش DragonClaw **لوحة كانبان** كل الأعمال على لوحة مهام واحدة:

- **تدفق متعدد الأعمدة** — عمود لكل وكيل؛ تتحرك المهمات داخل عمودها، ويظهر التعاون بين الوكلاء على الفور.
- **تحديث لحظي** — تُكتب نتائج المهام الفرعية والوكلاء الفرعيين الذين يُنشئهم الوكيل الرئيسي إلى البطاقات المقابلة فورًا.
- **تصوير الحالات** — تظهر حالات الخمول / التنفيذ / الخطأ / الاكتمال بعلامات مختلفة، فيبرز أي اختناق فورًا.
- **تحديث بنقرة واحدة** — يتيح زر «تحديث» في الشريط العلوي جلب آخر الحالات دون انتظار الدفع.

### 🧩 إدارة المهارات (Skill Management)

منظومة مهارات OpenClaw ضخمة، وتغلّفها DragonClaw في **مركز مهارات رسومي بالكامل** لتستخدمها دون أي إعداد يدوي:

- **تبويبات «موصى به / الكل / إدارة»** — بدِّل بينها من الأعلى، فيبدأ المبتدئ من الموصى بها ويستكشف الخبير في «الكل».
- **بحث بالكلمات المفتاحية** — يكفي كتابة كلمة في خانة البحث لتُصفّى المهارات، وتدخل الوسوم متعددة اللغات ضمن المطابقة.
- **تثبيت وإزالة بنقرة واحدة** — كل بطاقة مهارة مزوّدة بزرّ «تثبيت / إزالة»، والتراجع وإعادة التثبيت خفيفان بالمقدار نفسه.
- **بيانات المؤلف وآخر تحديث** — تعرض القائمة المؤلف وعدد التثبيتات وتاريخ آخر تحديث، لتقيّم نشاط الصيانة بنظرة واحدة.
- **مزامنة ذاكرة التخزين المؤقت المحلية** — تُخزَّن المهارات المُثبَّتة في ذاكرة التخزين المؤقت محليًا، فلا تحتاج إلى إعادة التنزيل في التشغيل التالي.

### 🖥️ مساعد الحاسوب (Computer Assistant)

بالتعاون مع OpenClaw Gateway، تضمّ DragonClaw كذلك وحدة **مساعد الحاسوب** التي تُحضر قدرات الذكاء الاصطناعي إلى جهازك:

- **تنظيف القرص** — يفحص بذكاء الملفات الكبيرة وذاكرة التخزين المؤقت والعناصر القابلة للتنظيف، ويُقدّر المساحة التي يمكن تحريرها.
- **ترتيب الحاسوب** — يَحصل الذكاء الاصطناعي على صورة شاملة عن جهازك ليساعد في تصنيف الملفات وترتيبها.
- **إدارة البرامج** — عرض موحَّد لكل التطبيقات المُثبَّتة، مع ترتيب حسب الحجم وآخر تحديث وحجم الاستخدام، وإزالة التطبيقات غير المرغوبة بنقرة واحدة.
- **تحديث الحالة** — يعيد زر «تحديث الحالة» في الشريط العلوي فحص الجهاز للحصول على أحدث حالة للبرامج والعتاد.

### ⚙️ الإعدادات الموحَّدة (Unified Settings)

تجمع DragonClaw جميع المعاملات القابلة للضبط في **لوحة إعدادات موحَّدة** واحدة، حتى لا تتناثر الإعدادات في عدة ملفات:

- **الأساسيات** — المنطقة الزمنية، تنسيق الوقت (24 / 12 ساعة)، فترة نبض القلب، الجلسات المستقلة، التحميل الساخن، وغيرها من السلوكيات الأساسية.
- **البوابة** — وضع التشغيل (محلي / عن بُعد)، عنوان الاتصال، Token / كلمة المرور وغيرها من معلومات اتصال Gateway.
- **الوكلاء** — القيم الافتراضية للوكلاء، قوالب الشخصية، مساحة العمل الافتراضية، وغيرها.
- **الجلسات** — سياسة الاحتفاظ بالسجل، عتبة ضغط الرسائل، مستوى التفكير الافتراضي، وغيرها.
- **الأدوات** — مفاتيح تفعيل قدرات المهام الفرعية وتنفيذ الأوامر وقراءة/كتابة الملفات وغيرها.
- **الأمان** — مصادقة الاتصال عن بُعد، إدارة Token، التأكيد الثنائي للعمليات الحساسة.
- **الحفظ وإعادة التحميل** — يطبّق زر «حفظ» في أسفل اليمين التغييرات، ويُسقط زر «إعادة تحميل» في أعلى اليمين التعديلات ويُعيد قراءة القيم.

## 🏗️ البنية المعمارية

يتبع DragonClaw نموذج العمليات المتعددة المعياري في Electron:

```
┌──────────────────┐   preload/IPC    ┌──────────────────┐   WebSocket RPC   ┌──────────────┐
│     Renderer     │ ◀──────────────▶ │   Main Process   │ ◀───────────────▶ │   Gateway    │
│  Vue 3 + Vite    │   (contextBridge)│   Electron       │                   │   OpenClaw   │
│  Arco Design     │                  │   Node.js APIs   │                   │              │
└──────────────────┘                  └──────────────────┘                   └──────────────┘
         ▲                                       ▲
         │                                       │
    User Interface                       Native OS APIs
                                    (file system, dialogs, etc.)
```

- **Renderer** — تطبيق Vue 3 أحادي الصفحة مبني على Vite و Arco Design Vue.
- **Main** — عملية Electron الرئيسية لإدارة النوافذ ومعالجات IPC والتكامل مع نظام التشغيل.
- **Preload** — طبقة `contextBridge` رفيعة تُقدّم للـ Renderer واجهة API مصغّرة ومُحكَمة الأنواع.
- **Gateway** — عملية OpenClaw Gateway التي تُوصَل عبر قناة WebSocket RPC.

## 🚀 بداية سريعة

### 1. استنساخ المستودع

```bash
git clone https://github.com/<your-org>/dragonclaw.git
cd dragonclaw
```

### 2. تثبيت الاعتماديات

```bash
# Recommended
pnpm install

# Or with npm
npm install

# Or with yarn
yarn install
```

### 3. التشغيل في وضع التطوير

```bash
pnpm electron:dev
```

> يُشغّل هذا الأمر خادم تطوير Vite (`:5177`) ونافذة Electron معًا.

### 4. بناء نسخة الإنتاج

```bash
# بناء للمنصة الحالية
pnpm dist

# بناء عميل Windows
pnpm build:win

# بناء عميل macOS
pnpm build:mac

# بناء عميل Linux
pnpm build:linux

# بناء عميل المنصات الثلاث دفعةً واحدة
pnpm build:all
```

توضع مخرجات البناء في المجلد `dist/`.

## 🛠️ السكربتات

| Command | Description |
| --- | --- |
| `pnpm start` | Launch Electron against an existing build |
| `pnpm dev` | Start the Vite dev server only |
| `pnpm build` | Build the renderer into `renderer/dist/` |
| `pnpm electron:dev` | Dev mode (Vite + Electron with HMR) |
| `pnpm dist` | Build a distributable installer for the current platform |
| `pnpm pack` | Build an unpacked directory (for local inspection) |
| `pnpm build:win` | Build a Windows installer |
| `pnpm build:mac` | Build a macOS installer |
| `pnpm build:linux` | Build a Linux installer |
| `pnpm build:all` | Build installers for all three platforms |

> جميع مديري الحزم `pnpm` و `npm` و `yarn` مدعومون. إذا كنت تستخدم `npm`، فاستبدِل `pnpm` بـ `npm run` (مثلًا `npm run electron:dev`)؛ وإن كنت تستخدم `yarn`، فاستبدِل `pnpm` بـ `yarn` (مثلًا `yarn electron:dev`).

## 📦 البناء والإصدار

تُنتَج المُثبّتات عبر [electron-builder](https://www.electron.build/):

| Platform | Target | Output |
| --- | --- | --- |
| Windows | NSIS | `dist/*.exe` installer |
| macOS | DMG / arm64 / x64 | `dist/*.dmg` |
| Linux | AppImage | `dist/*.AppImage` |

بيانات التطبيق الوصفية:

- `appId`: `com.dragonclaw.app`
- `productName`: `DragonClaw`
- أقل حجم للنافذة: `800 × 600`

> يتيح مُثبّت Windows اختيار مجلد التثبيت، وينشئ اختصارات على سطح المكتب وقائمة ابدأ تلقائيًا.

## 📁 هيكل المشروع

```
dragonclaw/
├── src/
│   ├── main/        # Electron main process (windows, IPC, database, services)
│   ├── preload/     # Preload bridge (contextBridge)
│   ├── renderer/    # Vue 3 + Arco Design renderer
│   │   ├── api/     #   Renderer-side API wrappers
│   │   ├── views/   #   Business pages (agent / skill / session …)
│   │   ├── components/
│   │   ├── core/    #   WebSocket / IPC primitives
│   │   ├── utils/
│   │   └── stores/
│   └── shared/      # Constants shared between main and renderer (e.g. IPC channel names)
├── doc/             # Gateway protocol documentation
├── build/           # Icons and brand assets
├── config.json
├── package.json
├── tsconfig.json
└── vite.config.ts
```
